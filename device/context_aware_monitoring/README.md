# Context Aware Monitoring Component

## Overview
This component provides context-aware monitoring for CTB bus safety systems. It uses computer vision and deep learning to detect potential hazards and provide real-time warnings.

**Optimised for Raspberry Pi deployment** — all YOLO models can run via ONNX Runtime (no heavy `ultralytics` needed at inference time).

## Features
- **Lane Detection**: YOLOv8 segmentation model for detecting road lanes
- **Lane Departure Warning**: Audio alerts when approaching solid lanes
- **Lane Memory Tracker**: Persists lane positions across frames when detection drops
- **Object Detection**: Detect vehicles and other road objects (ONNX or PyTorch)
- **Depth Estimation**: MiDaS ONNX model for distance measurement
- **Proximity Analysis**: Determine if objects are close, medium, or far
- **Driver Behaviour Analysis**: Traffic density, risky manoeuvre detection
- **Adaptive Processing**: Frame-skip, conditional MiDaS, ROI crop, resolution scaling
- **CTB Health Monitoring**: Integration with central monitoring system via MQTT

## Configuration

**All settings are read from `device/device_config.json`** — no interactive `input()` prompts.

Key config fields under `components.context_aware_monitoring`:

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `enabled` | bool | `true` | Enable/disable this component |
| `use_onnx` | bool | `true` | Prefer ONNX models over PyTorch |
| `enable_lane_detection` | bool | `true` | Enable lane segmentation |
| `enable_behavior_analysis` | bool | `true` | Enable driver behaviour analyser |
| `enable_adaptive_processing` | bool | `true` | Enable Pi frame-skip optimisation |
| `show_gui` | bool | `true` | Show OpenCV preview window |
| `video_source` | string | `"0"` | `"0"` = webcam, or file path e.g. `"dash_cam.mp4"` |
| `yolo_imgsz` | int | `384` | YOLO input image size (lower = faster) |
| `depth_input_size` | [int,int] | `[256,256]` | MiDaS input resolution |

Top-level fields also used: `vehicle_speed_kmh`, `camera.index`, `mqtt_*`, `device_key`.

### Example — use a video file instead of webcam

Edit `device/device_config.json`:
```json
{
  "components": {
    "context_aware_monitoring": {
      "video_source": "C:/videos/dash_cam.mp4"
    }
  }
}
```

## Converting Models to ONNX (for Raspberry Pi)

Run the export script **once on a development machine** (needs `ultralytics` + GPU):

```bash
cd device
python export_models_to_onnx.py          # exports all .pt → .onnx
python export_models_to_onnx.py --check  # list models without exporting
```

Then copy the generated `.onnx` files to the Pi. On the Pi you only need `onnxruntime` — no `ultralytics` or PyTorch required.

> **Note:** Lane segmentation still falls back to `.pt` for mask output when needed, since OnnxYOLO does not produce segmentation masks. Keep both `.pt` and `.onnx` on the device if lane detection is enabled.

## Usage

### Standalone (auto-configured from device_config.json)
```bash
cd device/context_aware_monitoring
python object_distance_measurement.py
```

### Via main orchestrator
```bash
cd device
python main.py   # starts all enabled components
```

## Integration with CTB System
This component integrates with the CTB bus monitoring system via MQTT:
- Sends health status updates
- Reports lane departure violations
- Reports wrong-side driving violations
- Sends proximity warnings for detected objects
- Reports driver behaviour violations (tailgating, harsh braking)
